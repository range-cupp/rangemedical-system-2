# Range Medical System — Complete Backend Architecture

## Overview

**App URL:** app.range-medical.com
**Stack:** Next.js 14 (Pages Router), Supabase (PostgreSQL + Storage), Vercel (hosting + cron)
**Styling:** Inline styles (no Tailwind/CSS modules)
**Key Integrations:** GoHighLevel CRM, Stripe Payments, Cal.com Booking, Resend Email, Twilio SMS

## Protocol Types & Configuration

All medication/dosage options live in `lib/protocol-config.js` (single source of truth, 670+ lines).

### Protocol Types
| Type | `program_type` | Tracking Method | Key Fields |
|------|----------------|-----------------|------------|
| HRT | `hrt` | Time-based (vial supply) | medication, hrt_type (male/female), supply_type (prefilled/vial), delivery_method, secondary_medication |
| Weight Loss | `weight_loss` | Time-based + injection count | medication (Semaglutide/Tirzepatide/Retatrutide), selected_dose, delivery_method (in_clinic/take_home), pickup_frequency |
| Peptide | `peptide` | Time-based (10/30/60/90 days) | medication (60+ options across 10 groups), dosage, frequency, cycle tracking |
| IV Therapy | `iv` | Session-based | total_sessions, sessions_used |
| HBOT | `hbot` | Session-based | total_sessions, sessions_used |
| Red Light | `rlt` | Session-based | total_sessions, sessions_used |
| Injection | `injection` | Session-based | total_sessions, sessions_used |
| Combo Membership | `combo_membership` | Creates 2 protocols (HBOT + RLT) | — |
| Labs | `labs` | — | panel_type |

### Medication Categories
- **Peptides (60+):** GH Secretagogue, Growth Hormone, Recovery, Weight Loss, GLP-1, Skin, Immune, Longevity, Bioregulators, Cognitive
- **Weight Loss:** Semaglutide, Tirzepatide, Retatrutide (each with dosing schedules)
- **HRT:** Testosterone Cypionate (male/female, vial/prefilled), with secondary medications (Gonadorelin, HCG, Nandrolone)
- **IV:** NAD+ (250/500/750/1000mg), specialty IVs
- **HBOT/RLT:** Session packs and memberships

## Key Data Flows

### 1. Protocol Creation (from POS purchase)
```
POS Charge (Stripe) → purchases table → autoCreateOrExtendProtocol() → protocols table → GHL sync (syncProtocolToGHL)
```
- `lib/auto-protocol.js` maps POS service categories → protocol types
- Auto-creates new protocol OR extends existing one (adds sessions, extends end_date)
- Combo memberships create 2 protocols (HBOT + RLT)

### 2. Protocol Creation (manual assignment)
```
Command Center assign form → POST /api/protocols/assign → protocols table → GHL sync → drip emails → guide SMS
```
- Handles duplicate prevention (checks for existing active protocol of same type)
- Sends consent form SMS, guide SMS, and starts drip email sequence for WL

### 3. Service Logging (single source of truth)
```
Service Log form → POST /api/service-log → service_logs table → GHL sync (syncSessionLogToGHL)
```
- 8 service types: Testosterone, Weight Loss, Vitamin, Peptide, IV, HBOT, Red Light, Supplement
- Dynamic form fields based on service type
- Decrements sessions_used or tracks injections

### 4. Lab Management Pipeline
```
Blood draw scheduled → lab results entered → lab_journeys stage progression → provider review → consult
```
- 5 stages: Blood Draw Complete → Results Received → Provider Reviewed → Consult Scheduled → Consult Complete
- Drag-to-advance UI in LabsPipelineTab

### 5. HRT Blood Draw Schedule
```
Protocol start_date → getHRTLabSchedule() → 5 scheduled draws (day 0, 56, 140, 224, 308)
```
- Matching priority: exact label match → date proximity (±28 days)
- Status: completed/upcoming/overdue

### 6. Communications
```
All SMS/email → logComm() → comms_log table
```
- SMS via: GHL API or Twilio
- Email via: Resend API
- All logged with channel, message_type, patient_id, status

## Cron Jobs (Vercel)

All scheduled in `/vercel.json`. Times shown in PT.

| Cron Job | Schedule (PT) | Purpose |
|----------|---------------|---------|
| `link-forms` | 10pm | Links unlinked consents/intakes to patients (by ghl_id → email → phone) |
| `complete-protocols` | 11pm | Auto-completes protocols past end_date (excludes WL and HRT) |
| `wl-midpoint-reminder` | 7am | Alerts clinic when monthly WL patient hits 2-week mark |
| `weekly-checkin-reminder` | 9am | Sends weekly WL check-in SMS on patient's injection day |
| `wl-drip-emails` | 6am | 4-email drip sequence for new WL patients (Resend) |
| `peptide-reminders` | 8am | 3 types: 10-day follow-up, 30-day weekly check-ins, 30-day re-up |
| `hrt-lab-reminders` | 9am | Consolidated SMS to staff about upcoming HRT lab draws (within 14 days) |
| `hrt-injection-reminders` | 9am | HRT take-home injection reminders (daily, mon_thu, or tue_fri schedule) |
| `injection-reminders` | 9am | General take-home injection reminders (respects weekly frequency) |
| `oxygen-daily` | 7am | Next day in 30-day oxygen email series (Resend) |
| `hrt-iv-reminders` | Manual | Reminds HRT patients about complimentary monthly IV |
| `generate-alerts` | Daily | Identifies at-risk patients (low completion, inactive >14d, score drops) |

### Cron Safeguards
- **Idempotency:** All reminders check `protocol_logs` table before sending
- **Time-gating:** Injection reminders only send 9am-6pm PST
- **Deduplication:** Consent/intake linking checks 5-minute windows
- **Limits:** Form linking capped at 500 records per run

## Architecture Patterns

1. **Single Source of Truth:** `service_logs` for all service deliveries, `protocol-config.js` for all medication/dosage options
2. **Protocol Lifecycle:** Created (from POS or manual) → Active → Completed (auto by cron or manual)
3. **GHL Sync:** Every protocol action syncs to GHL CRM (contact fields, notes, tags)
4. **Cron Idempotency:** All reminders check `protocol_logs` before sending to prevent double-sends
5. **Token-Based Access:** Patient portal and onboarding use unique tokens (no auth required)
6. **Webhook-Driven:** Consent, intake, appointment, and booking data flows into the system via webhooks
7. **Auto-Protocol:** POS purchases auto-create or extend protocols via `autoCreateOrExtendProtocol()`
8. **Communication Logging:** All automated SMS/email logged to `comms_log` via shared `logComm()` utility
